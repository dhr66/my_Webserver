使用状态机解析 HTTP 请求
状态机：按顺序解析 请求行 → 请求头 → 请求体
状态机就是：按顺。序处理每个部分，每处理完一部分就进入下一个状态


为什么需要线程池？
  - 创建/销毁线程很耗时
  - 线程池预先创建好线程，重复使用
  - 控制并发数量，避免资源耗尽

函数对象：std::function<void()> 可以存储任何可调用的对象（函数、lambda、绑定表达式）
,是一种数据类型，类似int
std::function<void()> task  ：可以存储任何"无参数、无返回值"的函数

  3. 线程分离 vs 线程结合

  // 分离线程（推荐）
  pthread_detach(thread_id);
  // 线程结束后自动释放资源，不需要 join

  // 结合线程
  pthread_join(thread_id, nullptr);
  // 阻塞等待线程结束，手动回收资源
